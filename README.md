# 空気の裏字幕 / IMPACT IN FACT

音声をリアルタイムで取り込み、字幕・発話意図・ファクトチェック・要約をひと続きで行う音声分析ダッシュボードです。

## 概要

動画や会議の音声を取り込み、以下をまとめて行います。

- 音声の文字起こし
- 発話ごとの意図推定
- 発言内容の外部検証候補の抽出
- 外部検索に基づく簡易的な裏取り
- セッション全体の要約と検証レポート生成
- 一部パネルのポップアップ分離表示

## 技術スタック

| 分類 | 内容 |
|------|------|
| フレームワーク | Next.js App Router |
| 言語 | TypeScript |
| UI | React + Tailwind CSS |
| 音声取得 | `MediaRecorder`, `getDisplayMedia`, `getUserMedia` |
| 文字起こし・LLM分析 | Sakura AI API |
| 外部検索 | Tavily API |

## セットアップ

### 1. 依存パッケージのインストール

```bash
npm install
```

### 2. 環境変数の設定

`.env.local` ファイルを作成し、以下を設定してください。

```env
SAKURA_AI_API_KEY=your_sakura_api_key
SAKURA_AI_BASE_URL=https://...
SAKURA_STT_MODEL=your_stt_model
SAKURA_CHAT_MODEL=your_chat_model
TAVILY_API_KEY=your_tavily_api_key  # 省略可（外部照合が無効になります）
```

### 3. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) を開きます。

## 使い方

### 1. 録音前の設定

- **コンテンツ種別**: YouTube動画・オンライン会議・セミナーなど
- **入力言語**: 自動・日本語・英語
- **分析メモ**: 何を重視して分析したいか（必須）
- **音声取得方法**: タブ音声またはマイク

> 分析メモが空の場合、録音を開始できません。

### 2. 録音中の表示

Start 後、以下がリアルタイムで更新されます。

- 直前の発話意図分析
- 検証アシスト（要確認かどうか）
- 通常字幕の履歴
- 発話分析の一覧
- 分析アシスタント（文脈について質問可能）

### 3. 停止後のレポート

Stop 後に「レポート生成」ボタンで以下を生成できます。

- セッション全体の要約
- トピック一覧
- 検証まとめ（支持・反証・未解決）
- Markdown形式でダウンロード

### 4. ポップアップ分離表示

右上メニューから以下を別ウィンドウで表示できます。

- リアルタイム字幕
- リアルタイム発話分析
- 検証アシスタント
- 分析アシスタント

## ディレクトリ構成

```
.
├── app/
│   ├── page.tsx              # アプリ本体（録音・状態管理・各種連携）
│   ├── layout.tsx
│   ├── popup/                # ポップアップ分離表示用ページ
│   └── api/
│       ├── transcribe/       # 音声認識（Sakura STT）
│       ├── infer/            # 発話意図分析
│       ├── verify/           # 外部検証（Tavily連携）
│       ├── report/           # セッションレポート生成
│       └── assistant/        # 分析アシスタント
├── components/               # UIコンポーネント
├── lib/                      # API連携・プロンプト・ユーティリティ
└── types/                    # 型定義
```

詳細なアーキテクチャと処理フローは [APP_EXPLANATION.md](./APP_EXPLANATION.md) を参照してください。

## 注意事項

- セッションデータはメモリ上のみで、ページを閉じると失われます
- 外部照合はTavilyの検索結果スニペットに基づくため、最終判断は必ず人手確認が必要です
- `TAVILY_API_KEY` がない場合は外部照合が「未実行」扱いになります
